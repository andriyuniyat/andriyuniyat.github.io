Atlassian administration and automation to reduce operational friction

Improving intake-to-resolution flow using Jira/Confluence administration, permissions, templates, and automation.

Context

Teams had inconsistent request intake, manual triage, and unclear ownership patterns. This created delays, rework, and avoidable escalations.

My role

  • Jira/Confluence administration: templates, permissions, workflow design, and operational standards
  • Partnered with support and stakeholders to define routing rules and clarity around ownership
  • Built documentation and “how-to” guidance to drive adoption

Approach

  • Standardized intake via request forms, templates, and required fields
  • Reduced manual triage using automation rules for routing, status updates, and notifications
  • Permission hygiene: aligned access to least-privilege needs and reduced ad-hoc admin work
  • Operational documentation: created runbooks and “what happens next” guides for requesters

Outcomes

  • Faster intake-to-assignment by removing manual handoffs
  • Improved request quality and reduced back-and-forth with standardized fields and guidance
  • Better accountability through clearer ownership and lifecycle checkpoints
  • Reduced repeat questions via concise documentation and reusable templates

What I’d do next

  • Introduce lightweight metrics (cycle time, reopen rate) to guide continuous improvement
  • Formalize a change log process to keep stakeholders aligned and reduce surprise changes

Note: This write-up is intentionally generalized and does not include internal project names, ticket keys, or proprietary workflows.
